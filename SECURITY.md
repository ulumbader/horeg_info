# Kebijakan Keamanan

## Versi yang didukung

Selama proyek belum memiliki rilis stabil, perbaikan keamanan hanya diterapkan pada branch default terbaru.

## Melaporkan kerentanan

Jangan membuka issue publik untuk kerentanan, credential yang terpapar, atau detail eksploitasi.

Gunakan fitur **Security > Report a vulnerability** pada repository GitHub. Maintainer harus mengaktifkan **Private vulnerability reporting** sebelum repository diumumkan ke publik.

Sertakan bila memungkinkan:

- komponen dan versi yang terdampak;
- langkah reproduksi minimal;
- dampak yang diperkirakan;
- bukti konsep yang tidak mengakses data pihak lain; dan
- saran mitigasi atau perbaikan.

Jangan menguji pada deployment production, mengakses data pengguna lain, melakukan denial of service, atau mempublikasikan temuan sebelum maintainer menyatakan perbaikan tersedia.

Maintainer akan mengonfirmasi penerimaan laporan, menilai dampaknya, dan mengoordinasikan waktu perbaikan serta publikasi secara privat.
